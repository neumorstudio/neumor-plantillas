# Instrucciones para Codex en este repo

Estas reglas aplican a cualquier agente Codex que trabaje en este proyecto.

## Reglas clave
- No tocar la base de datos salvo que sea estrictamente necesario.
- Si se necesita acceder a la BD, justificarlo y preferir primero las opciones no invasivas.
- Consultar siempre Supabase a traves del CLI para tener informacion actualizada.
- Al anadir nuevas webs, usar siempre el CLI del proyecto para crearlas.
- Respetar la estructura existente del proyecto (apps/, packages/, etc.) y los patrones actuales.

## Expectativas de trabajo
- Mantener los cambios acotados al objetivo solicitado.
- Evitar introducir archivos o flujos nuevos si no son necesarios.
- Si hay dudas sobre el flujo correcto, pedir confirmacion antes de proceder.
